# Interface Technical Reference (`Interface.lua`)

This document provides a comprehensive technical reference for the Yapper Settings UI. It explains the dynamic schema generation, widget lifecycle, and control registration mechanisms.

## Module State

The `Interface` module is designed to be fully dynamic, recreating and releasing UI components as the user navigates different categories.

| Field | Type | Description |
| :--- | :--- | :--- |
| `ActiveControls` | `table` | List of currently active widgets on the settings page. |
| `WidgetPool` | `table` | Pool of recycled frames (Checkboxes, Sliders, EditBoxes) for reuse. |
| `RenderCache` | `table` | Cached layout information used to prevent frame-thumping on screen resize. |
| `Categories` | `table` | Definition of sidebar sections and their associated paths in `Core.DEFAULTS`. |
| `MainWindow` | `Frame` | The root container for the settings UI. |
| `Launcher` | `Frame` | The custom minimap button. |

## UI Schema & Lifecycle

### `Interface:Init()`
Initialises the settings module, registers slash commands, and setups the minimap button.

### `Interface:BuildConfigUI()`
The entry point for generating the settings panel.
1.  **Clearance**: Calls `ClearConfigControls()` to return all active widgets to the pool.
2.  **Schema Walk**: Iterates the `CATEGORIES` and `DEFAULTS` tables.
3.  **Instantiation**: Calls specific `Create*Control` methods based on the data type in the schema.

### `Interface:SetSettingsChanged(bool)`
Marks that a setting affecting the visual overlay has been modified.
- Triggers a `Spellcheck:ApplyState` or `EditBox:ApplyConfigToLiveOverlay` call upon closing the window.

## Widget Management

### `Interface:AcquireWidget(widgetType, parent, template, frameType)`
Retrieves a widget from the pool or creates a new one if the pool is empty.
- `widgetType`: Descriptive name for the pool bucket (e.g., "CheckBox").

### `Interface:ReleaseWidget(widget)`
Returns a widget to the `WidgetPool`. Resets its scripts and hides it before storage.

### `Interface:AddControl(widget)`
Registers a widget as "Active," ensuring it will be released when the user switches categories or closes the window.

## Control Creation API

| Method | Component | Parameters |
| :--- | :--- | :--- |
| `CreateCheckBox` | Toggle | `(parent, label, path, cursor)` |
| `CreateSlider` | Range | `(parent, label, path, cursor)` |
| `CreateTextInput` | String | `(parent, label, path, cursor)` |
| `CreateColorPicker` | Colour | `(parent, label, path, cursor)` |
| `CreateDropdown` | Selection | `(parent, label, path, options, cursor)` |

### `LayoutCursor`
A helper object used during `BuildConfigUI` to manage vertical spacing and automatic indentation.
- `Y()`: Returns current relative Y offset.
- `Advance(amount)`: Moves the cursor down.
- `Pad(px)`: Adds fixed padding.

## Performance & Taint

### `Interface:PurgeRenderCache()`
Clears all cached layout offsets. Called when the Global UI scaling changes.

### `Interface:IsPathDisabledByTheme(path)`
Heuristic used to gray out settings that are currently being overridden by a Theme or a Blizzard Skin Proxy.
