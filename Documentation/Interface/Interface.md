# Interface & Settings UI (`Interface.lua`)

The `Interface` module manages the Yapper settings window (accessible via `/yapper` or the minimap button). It uses a dynamic schema-driven approach to build its UI components based on the `DEFAULTS` in `Core.lua`.

For a complete function-level API and widget-pooling reference, see the **[Interface Technical Reference](Reference.md)**.

## Module API

### `YapperTable.Interface`

| Method | Description |
| :--- | :--- |
| `Interface:Show()` | Displays the main settings window. |
| `Interface:Toggle()` | Toggles visibility of the settings window. |
| `Interface:SetSettingsChanged(bool)` | Marks that a refresh of the EditBox overlay is needed. |

## UI Construction

The interface is built using a custom "Schema" system. Instead of hardcoding every button, `Interface.lua` walks the `CATEGORIES` table and the `DEFAULTS` table to generate:
- **Toggles**: For booleans.
- **Sliders**: For numeric values with defined ranges.
- **EditBoxes**: For strings and file paths.
- **Colour Pickers**: For `{r, g, b, a}` tables.

### Categories
- **General**: Minimap, Spellcheck toggles, Sticky behaviour.
- **Appearance**: Themes, Colours, Font settings, Shadows.
- **Advanced**: Bridge toggles, history limits.
- **Diagnostics**: Real-time performance metrics (FPS, Memory, Event logs).

## Keyboard Interaction

### ESC Key Behaviour
The settings window is registered with `UISpecialFrames` (via `tinsert(UISpecialFrames, frameName)` in `CreateMainWindow`), so pressing ESC closes it — matching standard WoW panel behaviour.

**Focus trapping**: When a text input field within the settings is focused, pressing ESC or Enter calls `ClearFocus()` on the input rather than closing the window. This prevents accidental dismissal while editing values. A second ESC press (with no input focused) will close the window as expected.

### Widget Pool & Theme Overrides
- All dynamic controls are acquired from a widget pool via `AcquireWidget` and released on page switches via `ClearConfigControls`.
- `CreateResetButton` automatically registers its widget via `AddControl`; callers must NOT call `AddControl` on the returned button (this caused pool corruption in earlier versions).
- `IsPathDisabledByTheme(path)` checks both the active theme's `allow*` flags and the Blizzard Skin Proxy state. When a setting is disabled, the corresponding control, label, and reset button are grayed out.

## Minimap Button

Yapper includes a standalone minimap button implementation (in `Interface:SetupMinimapButton`) that does not require external libraries like LibDBIcon. 
- It supports dragging via a custom `OnUpdate` handler.
- Left-click: Toggle settings.
- Right-click: Open current draft (experimental).

## Taint & Lockdown
Like the `EditBox`, the Settings UI is non-protected. However, it listens for `PLAYER_REGEN_DISABLED` to gracefully handle transitions if any secure actions are pending (though the interface itself performs no secure actions).
