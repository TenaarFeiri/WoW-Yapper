# Post Chunking (`Chunking.lua`)

The `Chunking` module is responsible for breaking down long chat messages into fragments that fit within World of Warcraft's ~255-byte limit per message. It is "link-aware" and "colour-aware," ensuring that formatting codes and item links are never split across messages (which would break them).

## Module API

### `YapperTable.Chunking`

| Method | Description |
| :--- | :--- |
| `Chunking:Split(text, limit, ...)` | **Main Entry Point**. Takes a raw string and returns an ordered table of strings (chunks). |
| `Chunking:SetDelineators(delim, prefix)` | Configures the markers used to denote continuation (e.g. `»`). |

## Key Features

### 1. Link Preservation
The chunker uses a custom tokeniser to identify WoW escape sequences (e.g., `|Hitem:1234...|h[Item Name]|h|r`). It treats these as "atomic" units that cannot be cut. If a link doesn't fit in the remaining space of a chunk, it is pushed to the start of the next chunk.

### 2. Smart Boundary Detection
Instead of cutting mid-word, Yapper looks for the last space character within the allowed byte limit to perform a clean wrap. If no space is found (e.g., a very long URL), it performs a safe UTF-8 byte-aware cut to avoid corrupting multi-byte characters.

### 3. TotalRP 3 (TRP3) Continuity
If TRP3 is active, Yapper identifies and carries forward unclosed colour tokens (`|cff...`). 
- If a chunk ends with an active colour, Yapper automatically appends `|r` to "close" the colour for that chunk.
- It then automatically prepends the matching `|cff...` to the beginning of the *next* chunk.
- This prevents the "white text" bug typically seen in standard WoW message splitting.

### 4. Delineators
Yapper adds visual markers (suffix on the current chunk, prefix on the next) to indicate that a post is continuing. These are configurable in the Appearance menu.

## Dependencies

- **Config**: Reads `CHARACTER_LIMIT` and `DELINEATOR` settings.
- UTF-8 safe boundary detection is implemented inline inside `Chunking.lua` itself; it walks backwards from the byte limit to find a safe multi-byte character boundary without splitting a continuation byte.
