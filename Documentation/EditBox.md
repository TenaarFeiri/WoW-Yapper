# EditBox Overlay (`EditBox.lua`)

The `EditBox` module manages the visual overlay that replaces Blizzard's chat input. It is responsible for intercepting `Show()` calls on the default chat, rendering the custom input box, and handling local user input before passing it to the `Chat` orchestrator.

## Module API

### Overlay Lifecycle
These methods control the visibility and state of the Yapper overlay.

| Method | Description |
| :--- | :--- |
| `EditBox:Show(origEditBox)` | Hides the Blizzard EditBox and displays the Yapper overlay in its place. |
| `EditBox:Hide()` | Hides the Yapper overlay. |
| `EditBox:HandoffToBlizzard()` | Closes Yapper and forces the Blizzard EditBox to show (used during combat lockdowns). |

### Configuration & Appearance
| Method | Description |
| :--- | :--- |
| `EditBox:ApplyConfigToLiveOverlay(force)` | Refreshes all visuals (colours, patterns, shadows, borders) on the active overlay. |
| `EditBox:RefreshLabel()` | Recalculates the channel label (e.g. `[Say]:`) based on the current chat type. |
| `EditBox:AttachBlizzardSkinProxy(...)` | Clones textures from the Blizzard EditBox to mimic its appearance (used for the "Blizzard Skin" theme). |

### Interaction & Navigation
| Method | Description |
| :--- | :--- |
| `EditBox:NavigateHistory(direction)` | Recalls previous messages using Alt+ArrowUp/Down. |
| `EditBox:CycleChat(direction)` | Cycles through available chat channels (Say, Party, Raid, etc.). |
| `EditBox:AddReplyTarget(name, kind)` | Tracks recent whisper targets for quick reply cycling. |

### Integration Hooks
| Method | Description |
| :--- | :--- |
| `EditBox:HookAllChatFrames()` | Identifies and hooks all standard Blizzard chat editboxes to trigger Yapper. |
| `EditBox:SetOnSend(function)` | **REQUIRED**: Registers the callback invoked when the user presses Enter. |
| `EditBox:SetPreShowCheck(function)` | Registers a filter that can suppress the Yapper overlay (used by the Queue and WIM compatibility). |

## Dependencies

- **Spellcheck**: If `Spellcheck:OnCursorChanged` is registered, it is invoked via `HookScript` to track the caret.
- **Chat**: Receives the final processed text via the `SetOnSend` callback.
- **History**: Updates draft snapshots as the user types.

## Implementation Details

### Focus Model
Yapper's overlay supports an **unfocused persistence** mode:
- When focus leaves the overlay (e.g. clicking the game world), the overlay stays visible but keypresses propagate through to the game via `SetPropagateKeyboardInput(true)`. This allows WASD movement, ability usage, etc.
- Pressing **Enter** refocuses the overlay without losing the draft text. The Enter keypress is consumed by Blizzard's Show hook — it never fires `OnEnterPressed` on the overlay editbox.
- The state is tracked via `_overlayUnfocused` (boolean), reset on `Hide()` and `SetFocus()`.

### Graceful Lockdown Close
When a chat lockdown begins (combat start, M+ timer commencement) while the Yapper overlay is active:
- **If you haven't started typing:** Yapper hands off to Blizzard's secure editbox immediately so you're ready to use standard chat.
- **If you ARE mid-thought:** Yapper stays open and starts a **1.5-second idle timer**. Every letter you type resets this timer, giving you space to finish your message.
- **Wait mode:** Once you stop typing for 1.5 seconds, Yapper automatically saves your text as a draft, hides itself, and opens the Blizzard chat box so your progress isn't lost.
- **Smart Filtering:** The timer specifically listens for *physical* keystrokes. Background updates—like the spellchecker scanning your text or other addons synchronising data—are ignored, ensuring the UI doesn't "get stuck" open when you're actually idle.
- **Combat End:** If combat ends before the 1.5s timer fires, the handoff is cancelled and you can keep using Yapper normally.

**Key Technical Fields:**
- `_lockdownIdleTimer`: The active C_Timer handle for the 1.5s wait.
- `_lockdownHandedOff`: Tracking flag to prevent focus-fight loops.
- `_lockdownEventRunning`: Ensures the handoff logic only initialises once per lockdown event.

### Taint Avoidance
Yapper avoids the `SendChatMessage` taint introduced in WoW 12.0.0 by:
1.  Hiding the protected `ChatFrameNEditBox`.
2.  Rendering a custom, non-protected `EditBox` frame.
3.  Directly invoking `YapperTable.Chat:OnSend` which eventually reaches `C_ChatInfo.SendChatMessage`. 
    - *Note*: Blizzard only protects the *native* EditBox widget when it contains text; custom frames can still send messages if they haven't been part of a tainted execution chain.

### The Blink Isolation
To preserve the caret blink animation while spellcheck textures are active, Yapper uses a dedicated `UnderlineLayer` and detaches its measurement `FontString` from the UI parent to prevent layout invalidation loops.

### Underline Geometry
Underline and highlight textures are rendered using overlay-local coordinates and exact cursor/line-height measurements. This avoids visual drift when the text wraps to a new line or when UI scale changes affect the EditBox frame height.
