# Dispatch Router (`Router.lua`)

The `Router` is a lightweight abstraction layer for World of Warcraft's various chat sending functions. It resolves target names and manages the technical differences between various channel types (System vs. Battle.net vs. Communities).

## Module API

### `YapperTable.Router`

| Method | Description |
| :--- | :--- |
| `Router:Send(msg, chatType, language, target)` | **Main Entry Point**. Sends a single message. |
| `Router:Init()` | Hooks the raw `SendChatMessage` and `BNSendWhisper` functions for state tracking. |
| `Router:ResolveBnetTarget(target)` | Converts a friendly target name into a Battle.net Game Account ID if needed. |
| `Router:DetectCommunityChannel(channelTarget)` | Identifies if a channel target belongs to a Blizzard Community (C_Club). |

## Send Logic

When `Router:Send` is called, it branches based on `chatType`:

- **Battle.net Whispers (`BN_WHISPER`)**:
  - Automatically resolves BNet friend names to Presence IDs.
  - Invokes `BNSendWhisper`.
- **Community Messages (`CLUB`)**:
  - Uses `C_Club.SendMessage` after resolving the community ID from the target string.
- **Numbered Channels (`CHANNEL`)**:
  - Automatically handles target names like "1" or "General" to map to the correct channel index.
- **Normal Messages (Say, Party, Guild, etc.)**:
  - Invokes the standard `SendChatMessage`.

## Integration

The Router is used exclusively by `Chat:DirectSend` and `Queue:RawSend`. It is the last step in the Yapper pipeline before data is sent to the Blizzard servers.
