# Message History & Drafts (`History.lua`)

The `History` module handles persistent storage for sent messages (Alt+Arrow navigation) and real-time draft saving to prevent text loss during crashes or reloads.

## Module API

### `YapperTable.History`

| Method | Description |
| :--- | :--- |
| `History:AddChatHistory(text, type, target)` | Commits a successfully sent message to the persistent Alt+Arrow history. |
| `History:SaveDraft(editBox)` | Serialises the current text and cursor position into character-local storage. |
| `History:ClearDraft(editBox)` | Clears the active draft (called after a successful send). |
| `History:AddSnapshot(editBox)` | Creates a local undo/redo point. Used by the Spellchecker before applying suggestions. |
| `History:Undo(editBox)` / `History:Redo(editBox)` | Navigates the local per-session undo/redo stack (Ctrl+Z / Ctrl+Y). |

## Key Features

### 1. Persistence
Sent messages are saved to `YapperLocalHistory`. This allows you to recall messages sent hours (or days) ago, even across character reloads.

### 2. Draft Recovery
Yapper continuously monitors the overlay. If you are typing a long post and suddenly:
- Your game crashes.
- You accidentally press Escape (if `RecoverOnEscape` is true).
- You reload your UI.
...the next time you open Yapper, your text and cursor position will be restored exactly as you left them.

### 3. Encrypted / Secret Handling
If `YapperTable.Utils:IsSecret(text)` returns true (e.g., for certain passwords or protected RP commands), the History module will refuse to save that message to disk.

## Implementation Details

### Snapshots
While the character-wide History is persistent, the Undo/Redo snapshots are **session-only**. They are stored in memory and wiped when you close the game or logout. Yapper automatically takes a snapshot every few seconds of active typing and immediately before a spellcheck suggestion is applied.
