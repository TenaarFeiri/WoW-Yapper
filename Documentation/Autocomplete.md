# Autocomplete (`Autocomplete.lua`)

The `Autocomplete` module provides predictive word completion for the Yapper EditBox. As the user types, a ghost-text preview appears after the cursor showing the most likely completion. Pressing **Tab** commits it; any other keystroke refines or dismisses it.

## Overview

Autocomplete is intentionally separate from the spellchecker — it runs on every keystroke in the positive direction (suggesting what you *will* write), while the spellchecker runs in the negative direction (flagging what you *have* written incorrectly).

**Current scope**: horizontal overlay EditBox and multiline editor (both supported). Autocomplete now supports binding to the multiline editor via `Autocomplete:BindMultiline(mlEditBox)` and `Autocomplete:UnbindMultiline()`. The ghost FontString is reparented to the active EditBox and the caret hook is installed/restored per-EditBox to avoid leaks.

## Data Cascade (Tiers)

Suggestions are resolved by walking through sources in priority order. The first hit wins.

| Tier | Source | Strategy | Notes |
| :--- | :--- | :--- | :--- |
| **1** | YALLM personal lexicon | Linear scan of ≤ 2 000 freq entries | Matches cleaned (apostrophe-stripped) keys; highest-frequency prefix match wins |
| **1b** | User's custom dictionary (`AddedWords`) | Linear scan | Words explicitly added via "Add to dictionary" in the suggestion popup |
| **2** | Primary locale dictionary | Binary search on sorted `dict.words` | ~17 iterations for 130 k words — O(log N) |
| **2b** | Base dictionary fallback | Same binary search on `dict.extends` | Only reached when the locale is a delta (e.g. `enGB → enBase`) |

Tiers 2 and 2b also run an optional **phonetic broadening pass** when the user types away from the current suggestion (direction-change retry), collecting additional candidates via the shared phonetic hash index.

## Confidence Narrowing

The number of dictionary candidates collected scales with prefix length to balance recall and performance:

| Prefix length | Scan cap | Phonetic pass |
| :--- | :--- | :--- |
| 2–3 chars (SHORT) | 12 | No |
| 4–5 chars (MEDIUM) | 8 | Yes |
| 6+ chars (LONG) | 4 | Yes (tie-breaker only) |

## Capitalisation Mirroring

If the first byte of the typed prefix is an ASCII uppercase letter (A–Z), the returned suggestion is capitalised via `CapFirst()`. This handles:
- Sentence-start words (after a period + space).
- Consciously capitalised proper nouns and names.

The rule applies consistently across all tiers, so YALLM, custom words, and dictionary all behave the same way.

## Ghost Text

A non-interactive `FontString` is parented to the overlay EditBox, rendered in a muted colour (`0.55, 0.55, 0.55, 0.7`), and positioned to continue from the caret.

### Positioning
Ghost-text coordinates are taken from the `OnCursorChanged(self, x, y, w, h)` hook on the active EditBox. For the overlay EditBox `x` alone is sufficient (WoW provides a frame-relative, scroll-corrected X), but for the multiline EditBox both `x` and `y` are used so the ghost follows the caret across lines. The ghost `FontString` is parented to the active EditBox (not the outer container), so it naturally scrolls with the text. `PositionGhost()` anchors with `LEFT` for the overlay and `TOPLEFT` for multiline and applies `GHOST_CARET_PAD` horizontally to avoid the caret overlapping the first ghost character.

### Lifecycle

```
OnTextChanged → ExtractWordAtCursor → GetSuggestion → ShowGhost
Tab pressed   → OnTabPressed → commit → YALLM:RecordUsage
Type away     → isDirChange → YALLM:RecordRejection → retry broad → HideGhost
Overlay hides → OnOverlayHide → HideGhost
```

## Dismissal Learning (negBias)

When the user types away from an active suggestion — i.e. the current text no longer matches the suggestion's prefix — that is treated as a **soft rejection**:

1. `YALLM:RecordRejection(prefix, {suggestion})` is called once on the first diverging keystroke.
2. The rejection is stored in `YALLM.db.negBias` keyed as `Clean(prefix):Clean(suggestion)`.
3. On subsequent lookups, `ScoreCandidate` applies up to **−12 points** (3 per dismissal, capped at 4 dismissals) to repeatedly-rejected suggestions.

This prevents the autocomplete from nagging the user with unwanted suggestions they have already dismissed multiple times.

## Module API

### `Autocomplete:GetSuggestion(prefix, broad)`
Run the full tiered lookup and return the best completion string, or `nil`.
- `prefix`: The partial word at the cursor.
- `broad`: When `true`, forces maximum scan width and phonetic candidates (used on direction-change retry).
- Capitalisation of the result mirrors the prefix's first letter.

### `Autocomplete:SearchDictionary(words, phonetics, prefix, yallmFreq, yallmNeg, broad)`
Tier-2 search: collects prefix and phonetic candidates, scores each with `ScoreCandidate`, and returns the best word string.

### `Autocomplete:ExtractWordAtCursor(text, pos)`
Walks backwards from the cursor byte position to find the current token boundary (space / newline / tab). Returns `(word, wordStart)` or `(nil, nil)` when the token is shorter than `MIN_PREFIX_LEN` (2).

### `Autocomplete:OnTextChanged(editBox)`
Main keystroke handler. Extracts the current word, runs the suggestion cascade, and calls `ShowGhost` or `HideGhost`. Detects direction changes and fires dismissal recording.

### `Autocomplete:OnTabPressed(editBox)`
Commits the active suggestion by replacing the partial word in the EditBox and advancing the cursor. Returns `true` if consumed, `false` to fall through to channel cycling.

### `Autocomplete:ShowGhost(suggestion, prefix, textUpToCursor)`
Writes the suffix (suggestion minus prefix) into the ghost `FontString` and shows it.

### `Autocomplete:HideGhost()`
Hides the ghost `FontString` and clears all active state.

### `Autocomplete:IsEnabled()`
Returns `true` only when both `EditBox.AutocompleteEnabled` and `Spellcheck.Enabled` are `true` in config. Autocomplete requires the spellcheck dictionary as its data source.

### `Autocomplete:BindMultiline(mlEditBox)`
Bind the autocomplete ghost to a multiline EditBox. Called by `Multiline:Enter()`; the FontString is reparented and the caret hook is installed on the provided EditBox. This avoids creating new FontStrings per open/close and prevents widget leaks.

### `Autocomplete:UnbindMultiline()`
Restore autocomplete to the overlay: hides the ghost, restores any hooked cursor script on the previously bound EditBox, and switches the active editbox back to the overlay.

## Dependencies

- **YALLM**: Tier-1 source (`freq`, `negBias`), rejection recording, acceptance recording.
- **Spellcheck**: Dictionary data (`GetDictionary`, `GetUserDict`, `GetPhoneticHash`). Must be enabled.
- **EditBox**: `OverlayEdit` widget (ghost FontString parent, `OnCursorChanged` hook target).
