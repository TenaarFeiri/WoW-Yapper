# Dictionary Generation Pipeline

Yapper uses a pre-processing pipeline to convert raw word-lists (`.txt`) into high-performance, memory-efficient Lua tables. This process ensures 1:1 parity between the offline dictionary analysis and the real-time spellcheck algorithm.

## Workflow

1.  **Source Input**: Words live directly in `Src/Spellcheck/Dicts/en*.lua`. The generator reads those files, extracts every quoted word string from them, and uses that as its input corpus — there are no separate `.txt` word-list files.
2.  **Execution**: Run the Python generator from the repository root:
    ```bash
    python3 tools/generate_phonetic_dict.py
    ```
3.  **Output**: Rewrites `enBase.lua` (universal intersection) and each locale delta (`enUS.lua`, `enGB.lua`, etc.) in-place.

## Key Technical Concepts

### 1:1 Phonetic Parity
The Python generator implements the exact same `Simplified Metaphone` logic as `Spellcheck.lua`. This ensures that if the generator places "knight" in the `NT` hash bucket, the Lua engine will look in the `NT` bucket when the user types "nite".
- **Rules**: Consonant backbone extraction, silent letter groups (GHT, PH, etc.), and terminal normalisation.

### Universal Base vs. Regional Deltas
To save memory, Yapper does not store redundant words across multiple locales.
- **`enBase.lua`**: Contains the intersection of all supported English locales (~130k words).
- **Deltas**: Locales like `enGB` only contain words that are *not* in the base (e.g., "colour", "aluminium").
- **Inheritance**: At runtime, regional dictionaries use Lua metatables (`__index`) to fallback to the base dictionary for word and phonetic lookups.

### Lazy-Loading Chunks
The base dictionary is too large to load as a single Lua table without hitting Lua 5.1's constant-table limits or causing a significant frame-rate hitch.
- **Chunking**: The generator splits words into `getWords_N()` functions of up to **15,000 words each** and phonetics into `getPhonetics_N()` functions of up to 10,000 entries each.
- **Registration**: When the locale file is executed, `Spellcheck:RegisterDictionary` is called with a builder function that iterates each `getWords_N()` / `getPhonetics_N()` chunk and inserts them into the dictionary tables directly — no metatables or `loadstring` involved.

## Maintenance

When adding new words to the dictionary:
- Open the relevant `.lua` file in `Src/Spellcheck/Dicts/` (e.g. `enUS.lua`) and add each new word as a quoted string inside any `getWords_N` function body.
- Run the generator from the repository root:
  ```bash
  python3 tools/generate_phonetic_dict.py
  ```
  The script will re-derive the base/delta split, **re-sort all word lists alphabetically**, recalculate phonetic hashes, and rewrite the `.lua` files in-place.
- Reload the addon in-game (or use `/reload`) and confirm the new words are accepted by the spellcheck engine.
