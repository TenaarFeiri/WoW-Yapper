# Chat Orchestration (`Chat.lua`)

`Chat.lua` is the central orchestrator of Yapper's internal logic. It sits between the user interface (`EditBox.lua`) and the delivery systems (`Queue.lua`, `Router.lua`), directing the flow of text according to message length and channel type.

## Module API

### `YapperTable.Chat`

| Method | Description |
| :--- | :--- |
| `Chat:Init()` | Wires the components together. Connects `EditBox` events to `Chat` methods and initialises the Router and Queue. |
| `Chat:OnSend(text, chatType, language, target)` | **Main Entry Point**. Receives raw text from the overlay. |
| `Chat:DirectSend(msg, chatType, language, target)` | A helper that pipes a single string through the `Router`. |

## Handling Logics

When `Chat:OnSend` is invoked, it performs the following sequence:

### 1. Short Messages (<= 255 bytes)
If the message is under the limit, it is passed immediately to `DirectSend`.

### 2. Splittable Channels
If the message is long and the channel supports splitting (e.g., Guild, Party, Say):
- The `Chunking` module is called to split the text into an array of strings.
- If there's only one chunk (e.g., after whitespace trimming), it's sent directly.
- Otherwise, the chunks are handed off to the `Queue` system for throttled delivery.

### 3. Non-Splittable Channels
If the channel does NOT support splitting (e.g., Whispers, BNet Whispers):
- The message is **truncated** at the 255-byte limit.
- The truncated message is sent directly.
- *Note*: This prevents "overflow kicks" by the server which are more common on private message channels.

### 4. Adaptive Learning Hooks
Regardless of message length or delivery method, `Chat:OnSend` triggers a background learning cycle:
- **Usage Recording**: The `YALLM:RecordUsage` method is called to scan the text for "sane" words.
- **Biasing**: Frequency counts are incremented in the account-wide database to improve future ranking suggestions.

## Dependencies

### Internal
- `EditBox`: Provides the source text and event callback.
- `History`: Called to commit the final text to the persistent history cache.
- `Chunking`: Used to calculate post splits.
- `Queue`: Manages the multi-part delivery lifecycle.
- `Router`: Provides the abstracted WoW chat sending functions.

### Third-Party Compatibility
- **WIM (WoW Instant Messenger)**: `Chat:Init` includes a check to see if WIM has focus on a whisper. If so, Yapper's overlay is suppressed to avoid input contention.
- **LibGopher / CrossRP**: If the `GopherBridge` is active and busy, `Chat` will suppress new overlays and forward chunks directly to the bridge instead of using Yapper's internal queue.
