# YALLM: Yapper Adaptive Learning Language Model

YALLM is a lightweight, personalised language model that allows the Yapper spellcheck engine to adapt to a user's unique typing habits, vocabulary, and manual correction history. It provides a persistent scoring bias that helps resolve phonetic collisions and prioritises user-preferred variants over standard dictionary suggestions.

## Core Mechanics

### 1. Cumulative Learning
YALLM tracks the frequency of every word typed by the user (and accepted by the engine). This data is stored account-wide, allowing a user's vocabulary profile to persist across multiple characters.
- **Frequency Bonus**: Commonly used words receive a small, incremental scoring bonus.
- **Selection Bias**: Manual selections (where a user explicitly chooses a suggestion other than the top-ranked one) apply a significantly higher scoring weight to that specific candidate.

### 2. Scoring Weights
YALLM bonuses are injected into the [Spellcheck ranking loop](Spellcheck.md#scoring-heuristics) before the final candidate list is sorted.

| Metric | Bonus (Score Mod) | Description |
| :--- | :--- | :--- |
| **Selection Bias** | -2.50 per hit | High-gravity bonus applied when a user manually chooses a word. |
| **Frequency Bonus** | -0.15 per hit | Incremental bonus based on passive usage frequency. |
| **Correction Cap** | 10 Hits | Bonuses are capped at 10 occurrences to prevent a single typo from permanently dominating the dictionary. |

## Anti-Contamination (Sanity Heuristics)

To prevent the engine from "learning" keyboard smashes or phonetic noise (e.g., `asdfghj` or `noooowwwwww`), YALLM uses a defensive gatekeeper called `IsSaneWord` before recording any usage.

### `IsSaneWord(word)` Checks:
1.  **Consonant-Cluster Filter**: Rejects any word containing 7 or more consecutive consonants. (Note: Valid English words like "strengths" or "catchphrase" remain within the allowed 5-6 char limit).
2.  **Repeated Character Block**: Rejects words with 3 or more identical consecutive characters (e.g., `helllo`).
3.  **N-Gram Anchoring**: Verifies that the word shares at least one common pronounceable structure (bigram) with the [Base Dictionary](Spellcheck.md#phontic-engine-simplified-metaphone).

## Memory Management (LFU-lite Eviction)

YALLM maintains a hard cap of **2,000 learned words** per account to stay within the WoW client's memory budget.

### Eviction Strategy:
- **LFU-lite**: When the cap is reached, the engine performs a **50-item random scan** of the database.
- **Lowest-Value Purge**: It identifies and evicts the entry with the lowest combined frequency/selection score.
- **Retention**: High-frequency vocabulary and recent manual selections are naturally protected by the random-scan approach, ensuring the model retains relevant data without O(N) sorting overhead.

## Module API

### `YALLM:RecordUsage(text)`
Scans a sent message and increments the frequency count for all "sane" words.

### `YALLM:RecordSelection(typo, correction)`
Records a manual user selection, applying a high-weight bias to the chosen word for future instances of that specific typo.

### `YALLM:GetBonus(candidate, typo)`
Calculates the combined frequency and selection bonus for a candidate word.

## Persistence
Data is stored in `YapperDB.SpellcheckLearned` (Account-Wide `SavedVariables`).
 - **Normalisation**: All keys are stored in a vowel-neutral normalised format to ensure learning persists across minor vowel-based typos.
