# Spellcheck Technical Reference (`Spellcheck.lua`)

This document provides a comprehensive technical reference for the Yapper Spellcheck engine. It covers the internal state, public API, and lifecycle of the real-time misspelling detection system.

## Module State

The `Spellcheck` module maintains several internal buffers and state variables to ensure high performance and low GC pressure.

| Field | Type | Description |
| :--- | :--- | :--- |
| `Dictionaries` | `table` | Map of loaded locale dictionaries (`enGB`, `enUS`, etc.). |
| `UserDictCache` | `table` | Local cache of user-added and ignored words per locale. |
| `UnderlinePool` | `table` | Object pool for underline textures to minimise frame creation. |
| `Underlines` | `table` | List of active underline textures currently shown on the overlay. |
| `DictionaryBuilders` | `table` | Temporary storage for incrementally loaded dictionary chunks. |
| `_asyncLoaders` | `table` | Active `C_Timer` handles for background dictionary indexing. |
| `_suggestionCache` | `table` | LRU cache for recent word suggestions to avoid redundant scoring. |
| `_scratchBag` | `table` | Per-call reusable buffer for letter-count scoring. |
| `_scratchBigrams` | `table` | Per-call reusable buffer for bigram overlap scoring. |
| `YALLM` | `table` | The adaptive learning module instance. |

## Core API

### `Spellcheck:Bind(editBox, overlay)`
Links the spellchecker to a specific EditBox and its visual overlay.
- `editBox` (Frame): The Yapper input EditBox.
- `overlay` (Frame): The parent frame for underlines and the suggestion menu.

### `Spellcheck:LoadDictionary(locale)`
Initiates the asynchronous loading of a dictionary file.
- `locale` (string): The WoW locale code (e.g., "enUS").
- **Note**: If the dictionary is already loaded, it triggers an immediate refresh.

### `Spellcheck:RegisterDictionary(locale, data)`
Internal callback for dictionary module registration.
- Handles chunked metadata building (Phonetic, Prefix, and N-gram indices).
- Throttles execution via `C_Timer.After` to prevent frame-rate hitching.

### `Spellcheck:ReloadDictionaries()`
Fully purges the current state and re-initialises dictionaries from disk.

## Suggestion Engine

### `Spellcheck:GetSuggestions(word)`
The primary entry point for word ranking.
1.  **Normalisation**: Converts word to lowercase and strips non-word characters.
2.  **Phonetic Matching**: Retreives candidates sharing the same phonetic hash.
3.  **N-gram Refinement**: Filters candidates based on bigram overlap.
4.  **Edit Distance**: Computes Damerau-Levenshtein distance for the top ~300 candidates.
5.  **Scoring**: Applies weighted bonuses for keyboard proximity, locale variants, and phonetic complexity.
6.  **Adaptive Learning**: Injects personalised biases based on [YALLM](YALLM.md) frequency and correction history.

**Weights Used**:
- `editDistance`: 1.25
- `lenDiff`: 1.50
- `prefix`: 0.85
- `letterBag`: 0.45
- `bigram`: 0.65
- `kbProximity`: 0.35

### `Spellcheck:EditDistance(a, b, maxDist)`
Optimized Damerau-Levenshtein implementation with transposition support.
- Uses reusable row buffers (`_ed_prev`, `_ed_cur`) to avoid per-call allocations.

## Adaptive Learning (YALLM)

### `YALLM:Init()`
Initialises the learned word database from `YapperDB.SpellcheckLearned`.

### `YALLM:GetBonus(candidate, typo)`
Calculates the combined ranking bias for a suggestion.
- **Frequency**: Small incremental bonus for passive usage.
- **Selection**: High-gravity bonus for manual user corrections.

### `YALLM:RecordUsage(text)`
Scans outgoing chat for "sane" words to increment their frequency rank.

### `YALLM:EvictLeastUsed()`
Maintains the 2,000-word cap using a tiered LFU-lite random scan.

## UI Integration

### `Spellcheck:OnTextChanged(editBox, isUserInput)`
Hooked to the EditBox's `OnTextChanged` script.
- Sets a `_textChangedFlag` and schedules a refresh.

### `Spellcheck:UpdateUnderlines()`
Evaluates the visible text area and draws red underlines for misspellings.
- **Scan Window**: Clamped to `SCAN_RADIUS` (default 1000 chars) around the cursor.
- **Blink Isolation**: Textures are anchored to a dedicated layer to prevent cursor-blink resets.

### `Spellcheck:ShowSuggestions()`
Positions and renders the suggestion menu.
- Automatically handles horizontal scroll offsets via `GetScrollOffset()`.
