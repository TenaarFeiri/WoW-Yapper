# Real-Time Spellcheck (`Spellcheck.lua`)

The `Spellcheck` module provides live misspelling detection and suggestion popups for the Yapper EditBox. It uses a high-performance phonetic and literal matching engine designed for the strict memory and CPU constraints of the World of Warcraft client.

## Core Architecture

For a complete function-level API and state reference, see the **[Spellcheck Technical Reference](Reference.md)**.

### Phonetic Engine (Simplified Metaphone)
To achieve relevance even with "typing by sound," Yapper uses a custom phonetic hashing algorithm identical to its offline dictionary generator.
- **Vowel Stripping**: Vowels are stripped after the first character to form a consonant backbone.
- **Silent Groups**: Specifically handles `GHT`, `PH`, `KN`, `GN`, and `WR`.
- **Phonetic Normalisation**: Normalises `CH`, `SH`, `C`, `Q`, `X`, and `Z` into a unified sounding set.
- **Punctuation Stripping**: All non-alphabetic characters (including apostrophes) are stripped *before* hashing to ensure consistency across spelling variations (e.g., `zhukov'` hashes the same as `Zhukov`).
- **Terminals**: Converts terminal `GH` to `F` (e.g., "tough").

### Chunked Dictionary Loading
To avoid massive memory spikes, the 130k+ word universal base dictionary (`enBase.lua`) uses a lazy-loading metatable architecture:
- **Index-to-Chunk Mapping**: Words are divided into 1000-word chunks.
- **On-Demand Hydration**: Chunks are only hydrated (deserialized into Lua tables) when a word ID in that range is first requested.

### Regional Delta Inheritance
Localization is achieved through lightweight regional deltas:
- `enUS` and `enGB` only contain words specific to their locale.
- They extend `enBase` via metatable `__index` linkages, ensuring a seamless lookup experience without duplicating the 130k universal words in memory.

### YALLM (Adaptive Learning)
Yapper includes a personalised language model ([YALLM](YALLM.md)) that learns from user behaviour:
- **Usage Frequency**: Common words gain ranking bonuses over time.
- **Manual Corrections**: Choosing a non-top suggestion prioritises that selection for future collisions.
- **Sanity Heuristics**: Anti-contamination filters prevent the engine from learning "keyboard smash" gibberish.

## Scoring Heuristics

Yapper ranks candidates using a weighted multi-factor scoring system:

| Factor | Weight | Description |
| :--- | :--- | :--- |
| **Phonetic Bonus** | -7.0 | Fixed bonus for words sharing the same sounding hash. |
| **Complexity Bonus** | -0.75 / char | Favours longer "proper" spellings over short phonetic noise if the sounding matches. |
| **Edit Distance** | 1.25 | Traditional Damerau-Levenshtein distance multiplier. |
| **Keyboard Proximity** | 0.35 | Bonus for "fat-finger" errors (adjacent keys on QWERTY/QWERTZ/AZERTY). |
| **Bigram/Bag** | Variable | N-gram and character-set intersection scores for tie-breaking. |
| **YALLM Bonus** | -0.15 to -2.5 | Personalised bonus based on usage frequency and manual selections. |

## Performance Optimisation

- **Scan Windows**: Only scans ~2000 characters around the caret in long messages.
- **Dynamic Capping**: Literal match searching is throttled at 1000 candidates for standard inputs to maintain 60fps.
- **Dictionary Doubling Prevention**: The engine explicitly detects pre-filled builder data to avoid redundant word insertions, preventing 2x memory spikes during dictionary reloads.

## Hidden/Advanced Settings

These settings are not exposed in the UI but can be modified via command line or `YapperLocalConf`.

- **N-gram Vocabulary Cap (`Spellcheck.NgramKeyCapSize`)**: The number of words indexed in the n-gram engine can be capped (default 0/uncapped) to limit memory overhead.
- **Lazy Phonetic Bucketing**: Categorization of large phonetic groups is capped at 5000 entries to prevent O(N) scaling hangs on massive sounding buckets.
- **Linear Metadata Purge**: Metadata eviction uses a fast linear-scan approach instead of O(N log N) sorting, ensuring steady performance during long sessions.
- **Fast Variant Rules**: Uses table-based prefix/suffix lookups for regional spelling differences (`or` vs `our`) to avoid `string.gsub` overhead in hot loops.
- **Parented Measuring**: Measurement `FontString` widgets are parented to the Overlay frame to ensure perfect scale inheritance and coordinate parity with the EditBox.
- **Cursor Idle Polling**: Text measurements are cached by cursor position, skipping redundant processing when the caret is stationary.
