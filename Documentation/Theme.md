# Visual Themes (`Theme.lua`)

Yapper supports a lightweight theme engine that allows the EditBox overlay to be styled with different presets. Themes can control colours, fonts, and enforce specific visual styles that cannot be overridden by the user.

## Module API

### `YapperTable.Theme`

| Method | Description |
| :--- | :--- |
| `Theme:RegisterTheme(name, data)` | Registers a new theme table into the global registry. |
| `Theme:SetTheme(name)` | Activates a theme. This copies basic colour values into the character config for immediate use. |
| `Theme:ApplyToFrame(frame, name)` | Applies specific font settings and executes the theme's `OnApply` hook. |

## Theme Schema

A theme table can contain the following fields:
- `name` (string): Unique identifier.
- `inputBg` (table): Default colour for the text input area.
- `labelBg` (table): Default colour for the channel label area.
- `textColor` (table): Default colour for typed text.
- `borderColor` (table): Default border colour (only used if `border = true`).
- `font` (table): `{path, size, flags}` block.
- `allowRoundedCorners` (bool): If `false`, Yapper will force flat corners and disable the user toggle in the UI. Default: `true`.
- `allowDropShadow` (bool): If `false`, Yapper will disable shadows and hide the user toggle in the UI. Default: `true`.
- `channelTextColors` (table): Map of chat channel keys (e.g., `SAY`, `PARTY`) to colour tables.
- `OnApply` (function): **Optional**. A script that is executed whenever the theme is applied to a frame. This can be used to add custom textures, animations, or border effects.

## Visual Overrides

Themes can enforce visual consistency by "locking down" certain settings.

### Enforced Styles
Fields like `allowRoundedCorners` and `allowDropShadow` act as hard constraints. 
- **Rendering**: Even if the user has these settings enabled in their local configuration, the rendering layer will ignore them if the theme explicitly forbids them.
- **Interface**: The corresponding toggles and Reset buttons in the Yapper settings interface will be disabled and labeled as `(Disabled by theme)`.

### Blizzard Skin Proxy
The "Use Blizzard skin proxy" setting acts as a high-priority system-level override. When active:
- It automatically suppresses rounded corners and drop shadows (equivalent to setting the `allow*` flags to `false`).
- Interface labels will display `(Disabled by Blizzard Skin)`.

## Standard Themes

### Yapper Default
The built-in flat-colour theme. It prioritises readability and high contrast. Now allows both rounded corners and shadows by default in the core schema.

### Blizzard Skin (`Src/Themes/Blizzard.lua`)
A special theme that uses `EditBox:AttachBlizzardSkinProxy` to clone the look of the default WoW chat boxes. It effectively "skins" Yapper to match the rest of the Blizzard UI.

## Theme Configuration
When a theme is selected via `Theme:SetTheme`, its colour values are written to `YapperLocalConf.EditBox`. However, if the user has manually changed a colour using the Yapper Interface colour pickers, those manual overrides are preserved and will "block" the theme's default colours for that specific field.
