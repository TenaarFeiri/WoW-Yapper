Migration Guide — Integrations & Addons
=======================================

Overview
--------
This guide helps addon authors and bridge maintainers adapt to Yapper's
refactor: the new `Src/API.lua` public API and the movement of bridge
integration points into documented filters and callbacks.

Goals
- Replace fragile direct callsites with stable, documented APIs.
- Ensure third-party bridges behave consistently and avoid duplicate signals.
- Provide concrete examples for common migration tasks.

Checklist
---------
1. Ensure the Yapper API is available

   Guard for `_G.YapperAPI` in your initialization code. The main Yapper
   TOC loads `Src/API.lua` early, but load-order variations are possible.

   ```lua
   if not _G.YapperAPI then
       -- Defer registration until PLAYER_ENTERING_WORLD or a safe event
   end
   ```

2. Replace direct EditBox/Chat calls with API registrations

   Typical direct calls to replace:
   - `YapperTable.TypingTrackerBridge:OnOverlayFocusGained(chatType)`
   - `YapperTable.TypingTrackerBridge:OnOverlayFocusLost()`
   - `YapperTable.RPPrefixBridge:ApplyPrefix(text, chatType)` (was called from Chat:OnSend)

   Replacement patterns:

   - Typing tracker -> overlay show/hide:

   ```lua
   _G.YapperAPI:RegisterCallback("EDITBOX_SHOW", function(chatType, target)
       MyBridge:SignalTyping(chatType)
   end)

   _G.YapperAPI:RegisterCallback("EDITBOX_HIDE", function()
       MyBridge:SignalStopTyping()
   end)
   ```

   - Prefix injection -> PRE_SEND filter:

   ```lua
   _G.YapperAPI:RegisterFilter("PRE_SEND", function(p)
       p.text = MyBridge:ApplyPrefix(p.text, p.chatType)
       return p
   end, 20) -- priority 20 so consumer filters (default 10) run first
   ```

3. Respect priorities

   - Filters use numeric priorities; lower numbers run earlier.
   - Choose a priority that places your bridge before/after consumer filters
     depending on intended behaviour.

4. Avoid double-calls

   - Remove legacy direct callsites in your fork of Yapper, or ensure you
     register via the API and do not keep inlined calls inside core modules.

5. Test in combat lockdown and channel edge-cases

   - Some bridges interact with secure paths or channel APIs (BNet, CLUB,
     CHANNEL). Test in group/raid/bnet modes and confirm no tainting occurs.
