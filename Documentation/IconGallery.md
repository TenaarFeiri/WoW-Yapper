# Icon Gallery (`IconGallery.lua`)

The `IconGallery` module provides the raid-target icon picker that appears
when a user types `{` in the overlay edit box.  It is a lightweight popup
grid showing all eight WoW raid icons, filterable by name or shorthand code.
Selecting an icon inserts the corresponding `{name}` tag at the cursor
position, which the EditBox or Multiline editor then renders as the icon.

## Triggering Behaviour

| Input | Result |
| :--- | :--- |
| Type `{` alone | Gallery opens showing all 8 icons. |
| Type `{mo` | Gallery filters; only `moon` remains visible. |
| Type `{rt5` | Gallery filters by shorthand code; only `moon` remains. |
| Select icon (click or keyboard) | `{name}` tag inserted at cursor; gallery closes. |
| `Esc` while gallery is open | Gallery closes; typed text left unchanged. |

## Module API

### `YapperTable.IconGallery`

| Method | Description |
| :--- | :--- |
| `IconGallery:Show(editBox, anchorFrame, query)` | Shows the gallery popup linked to `editBox`, anchored to `anchorFrame`, optionally pre-filtered by `query`. |
| `IconGallery:Hide()` | Hides the gallery and clears internal state. |
| `IconGallery:Filter(query)` | Updates cell visibility to match `query` (name or code prefix). |
| `IconGallery:Select(index)` | Inserts the icon tag for `index` (1-8) into the linked EditBox and hides the gallery. |
| `IconGallery:HandleKeyDown(key)` | Should be called from an `OnKeyDown` handler. Returns `true` if the key was consumed. Handles `Esc`, digit shortcuts 1-8, `Enter`, and `Tab`. |
| `IconGallery:OnTextChanged(rawEditBox, anchorFrame)` | Convenience driver. Call from an `OnTextChanged` hook; automatically shows, updates, or hides the gallery based on the text before the cursor. |
| `IconGallery:_GetIconMeta(index)` | Returns `{ index, text, code }` for icon `index` (1-8). Intended for use by the public API; the texture coords are considered internal. |

## Keyboard Shortcuts

When the gallery is open:

- **1-8** — Select the corresponding (enabled) icon immediately.
- **Enter / Tab** — Select the first enabled icon.
- **Esc** — Close without selecting.

The digit keys are consumed so the number does not appear in the edit box.

## Integration with External Addons

External addons can attach the gallery to their own `EditBox` widgets via the
public `YapperAPI` surface.  See [API.md](API.md) for full signatures and
examples.

### Quick start

```lua
-- Show the gallery on your own editbox
if YapperAPI and YapperAPI.ShowIconGallery then
    local myBox    = MyAddon.InputBox       -- a raw WoW EditBox frame
    local myAnchor = MyAddon.InputFrame     -- frame to anchor the popup to
    YapperAPI:ShowIconGallery(myBox, myAnchor)
end

-- Wire OnTextChanged so the gallery auto-appears/updates
myBox:HookScript("OnTextChanged", function(box)
    if YapperAPI and YapperAPI.ShowIconGallery then
        -- Drive the gallery the same way Yapper does internally
        local ig = YapperTable and YapperTable.IconGallery
        if ig then ig:OnTextChanged(box, myAnchor) end
    end
end)

-- Wire OnKeyDown so keyboard shortcuts work
myBox:HookScript("OnKeyDown", function(box, key)
    local ig = YapperTable and YapperTable.IconGallery
    if ig and ig:HandleKeyDown(key) then
        box:SetPropagateKeyboardInput(false)
    end
end)

-- React when an icon is picked
YapperAPI:RegisterCallback("ICON_GALLERY_SELECT", function(index, text, code)
    print(("Icon picked: %s (rt%d / %s)"):format(text, index, code))
end)
```

## Internal Notes

- The gallery frame (`YapperContinueFrame`) is created lazily on first `Show`.
- The popup anchors `BOTTOMLEFT → TOPLEFT` of the anchor frame with a 4-pixel gap.
- Disabled (filtered-out) cells are shown at 20 % opacity and cannot be clicked.
- `_suppressNextChar` / `_suppressChar` fields prevent the digit key character
  from being appended to the EditBox after a keyboard-shortcut selection.
